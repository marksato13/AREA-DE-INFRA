4Descripción del Proyecto
El presente proyecto tiene como propósito diseñar e implementar una infraestructura tecnológica híbrida y segura para Franco's SAC, empresa del sector agroindustria alimentaria, que permita soportar su proceso de transformación digital mediante la adopción de un ERP empresarial basado en microservicios y una red corporativa estructurada en tres locales.

Franco's SAC opera actualmente sin servidores, con una red WiFi no segmentada y sin sistemas de gestión integrados. El proyecto diseña e implementa una arquitectura híbrida on-premises + AWS Cloud sobre los hipervisores ESXi de los laboratorios Cisco de la UPeU, con los siguientes componentes principales:

Capa de red: segmentación por VLANs (Admin, Usuarios, DMZ, Servicios, Datos, Gestión, MGMT) en los 3 locales; Dual-WAN Failover con pfSense 2.7; túneles IPSec IKEv2 AES-256-GCM en topología hub-and-spoke (L1↔L2, L2↔L3, L2↔AWS); WAF NGINX + ModSecurity en DMZ; switch físico Cisco Catalyst 1000 con trunking 802.1Q.

Capa de servicios centralizados (Sede Central — Local 2): 4 hipervisores ESXi; Active Directory Samba (DC1 + DC2 réplica); NTP Chrony; File Server Samba/NFS; monitoreo Zabbix + Grafana + CloudWatch Agent; API Gateway NestJS en DMZ; NATS 2.x + JetStream como message broker; microservicios NestJS Docker (auth-ms, person-ms, sales-invent-ms, strategic-ms); bases de datos PostgreSQL 15 (person-db, sales-invent-db, strategic-db); streaming replication H3→H4 con RPO < 5 min.

Capa de sedes remotas (Local 1 y Local 3): pfSense Dual-WAN + IPSec por sede; RODC Samba (DC3 en L1, DC4 en L3) para autenticación local sin VPN; File Server local + Agente Zabbix.

Capa cloud AWS: Route 53 (DNS + health checks), CloudFront + S3 (Angular SPA CDN), AWS VPN Gateway (IPSec site-to-site), S3 Backups (pg_dump diario), ECR (imágenes Docker).

Todo bajo estándares internacionales ISO 27001, ISO 27005, NIST CSF, TIA/EIA-568, IEEE 802.x, ISO/IEC 11801, Uptime Institute, TIA-942, ASHRAE TC9.9, ISO/IEC 20000 e ITIL 4.
5Alcance del Proyecto

Incluye:

  • E1 — Diseño de Red: topología lógica y física de los 3 locales; segmentación por VLANs (10 Admin, 20 Usuarios, 30 DMZ, 40 Servicios, 50 Datos, 60 Gestión, 99 MGMT); diseño Dual-WAN + VPN IPSec IKEv2 mesh L1↔L2↔L3↔AWS; redundancia y alta disponibilidad; cumplimiento TIA/EIA-568, IEEE 802.x, ISO/IEC 11801
  • E2 — Planificación de Seguridad: identificación de activos críticos; análisis de riesgos (ISO 27005 / NIST CSF); políticas de seguridad; roles y responsabilidades (RACI); SGSI bajo ISO 27001
  • E3 — Diseño de Centro de Datos: arquitectura Tier II (Uptime Institute / TIA-942); layout físico 4 hipervisores ESXi Sede Central + H5 L1 + H6 L3; dimensionamiento CPU/RAM/almacenamiento; virtualización ESXi + arquitectura cloud híbrida AWS
  • Infraestructura de red: pfSense 2.7 Dual-WAN Failover por sede; Cisco Catalyst 1000 trunk 802.1Q; WAF NGINX + ModSecurity en DMZ; AP WiFi VLAN 20; túneles IPSec IKEv2 AES-256-GCM
  • Servicios centralizados (Sede Central): Active Directory Samba DC1 + DC2 réplica (H4); NTP Chrony; File Server Samba/NFS; RODC DC3 en L1 y DC4 en L3
  • Backend ERP — microservicios: API Gateway NestJS en DMZ; NATS 2.x + JetStream broker; auth-ms, person-ms, sales-invent-ms, strategic-ms (NestJS Docker); PostgreSQL 15 (person-db, sales-invent-db, strategic-db) en VLAN 50; streaming replication H3→H4 RPO < 5 min
  • Monitoreo híbrido: Zabbix 6.x + Grafana (on-prem completo via agentes y SNMP); CloudWatch Agent (KPIs correlacionados con AWS); dashboards operativo y ejecutivo
  • Cloud AWS: Route 53 + CloudFront + S3 (Angular SPA); AWS VPN Gateway site-to-site; S3 Backups pg_dump diario; ECR imágenes Docker microservicios
  • E4–E6 Semestre 2: implementación y testing de red en labs Cisco UPeU; implementación controles seguridad (IAM, cifrado, firewall, backup); operación centro de datos (SLA, monitoreo, procedimientos ITIL 4)
  • Documentación técnica formal bajo estándares internacionales y Código de Ética ACM

No incluye:

  • Adquisición e instalación de hardware físico en las instalaciones de Franco's SAC (el proyecto se implementa en labs ESXi de la UPeU)
  • Desarrollo del frontend Angular 20 ni del código fuente de los microservicios (responsabilidad del área SW)
  • Migración de datos históricos de la empresa al nuevo sistema
  • Capacitación al personal de Franco's SAC
  • Adquisición de licencias AWS en producción (el proyecto usa Free Tier o cuentas académicas)
